## Getting Started

### What is AfriThrive Global Admin System?

This is the administrative platform for managing AfriThrive Global's operations, including:
- Job postings and applications
- Scholarship programs and applications
- Language courses and enrollments
- Payment tracking
- User management
- Partner organizations

### Who Can Use This System?

This system is for authorized administrators and staff members who have been provided with login credentials by the AfriThrive Global IT team.

## Login Process

### Step 1: Access the System

1. Open your web browser (Chrome, Firefox, Safari, or Edge)
2. Navigate to the AfriThrive Global Admin portal URL (provided by your IT department)
3. You'll see the login screen with the AfriThrive logo

### Step 2: Enter Your Credentials

1. **Email/Username**: Enter the email address or username provided to you
2. **Password**: Enter your password (what you type will be hidden for security)
3. Click the **"Sign In"** button

### Step 3: First Time Login

If this is your first time logging in:
- You may be asked to change your password
- Follow the on-screen instructions to set a new, secure password
- Your password should be at least 6 characters long

### Forgot Password?

If you forgot your password:
1. Click the **"Forgot Password?"** link on the login page
2. Enter your email address
3. Check your email for a password reset link
4. Follow the instructions in the email to reset your password

### Login Issues?

If you cannot log in:
- ‚úÖ Check that CAPS LOCK is not turned on
- ‚úÖ Verify you're using the correct email/username
- ‚úÖ Make sure your internet connection is working
- ‚úÖ Contact your system administrator for help

## Course Timetables

### Overview

Create and manage course schedules for language programs.

### Viewing Timetables

1. Click **"Course Timetables"** in the sidebar
2. See all timetable sessions with:
   - Session name (e.g., "Spring 2026 Session")
   - Registration period
   - Class start date
   - Status (Active/Inactive)

### Creating a New Timetable

**Step 1: Create the Timetable**
1. Click **"+ Create Timetable"** button
2. Enter:
   - **Session Name**: e.g., "Spring 2026"
   - **Registration Start**: When students can start registering
   - **Registration End**: Last day to register
   - **Classes Start**: First day of classes
   - **Set as Active**: Check if this is the current session

3. Click **"Create Timetable"**

**Step 2: Configure Levels**
1. Find your timetable in the list
2. Click **three dots (‚ãÆ)** ‚Üí **"Manage Levels"**
3. For each level (A1, A2, B1, B2, C1, C2), set:
   - Duration in weeks
   - Total number of sessions
   - Total hours
4. Click **"Save Configurations"**

**Step 3: Set Up Tracks (Schedules)**
1. Click **three dots (‚ãÆ)** ‚Üí **"Manage Tracks"**
2. Create schedule options (e.g., Morning, Evening, Weekend)
3. For each track:
   - Track name
   - Days and times (e.g., Mon/Wed/Fri 9:00 AM - 11:00 AM)
4. Click **"Save Tracks"**

**Step 4: Generate Course Offerings**
1. Click **three dots (‚ãÆ)** ‚Üí **"Generate Offerings"**
2. Select languages to offer (French, Spanish, German, etc.)
3. Set maximum students per class
4. Click **"Generate Offerings"**
   - System automatically creates all combinations (Language √ó Level √ó Track)

### Managing Timetables

- **Edit**: Update dates or details
- **Activate**: Set as current active session (only one can be active)
- **Delete**: Remove a timetable (warning: deletes all related data)

## Frequently Asked Questions (FAQ)

### General Questions

**Q: How often should I change my password?**
A: We recommend changing it every 90 days for security.

**Q: Can I access the system from home?**
A: Yes, if you have internet access and your login credentials.

**Q: What if I forget my password?**
A: Use the "Forgot Password" link on the login page, or contact your administrator.

**Q: Can multiple people use the same login?**
A: No, each person must have their own account for security and accountability.

### Payment Questions

**Q: How do I know if a payment was successful?**
A: The payment status will show "Success" in green, and you'll see it in Payment History.

**Q: Can I refund a payment?**
A: Contact your system administrator or finance team for refund requests.

**Q: What if a student says they paid but it's not showing?**
A: Check Payment History with the student's email or reference number. If not found, contact support.

### Course Management

**Q: Can I edit a course after students have enrolled?**
A: Yes, but major changes (price, schedule) may affect enrolled students. Make changes carefully.

**Q: How do I know if a student is eligible for classes?**
A: Check Course Fees section. Students who paid ‚â•50% are marked as "Eligible."

**Q: Can I create custom course levels?**
A: No, the system uses standard CEFR levels (A1-C2). Contact support if you need custom levels.

### User Management

**Q: How many users can I add?**
A: There's no limit, but each user needs appropriate permissions.

**Q: Can I create custom roles?**
A: Yes, go to Roles & Permissions to create new roles with specific permissions.

**Q: What happens to a deactivated user's data?**
A: The data remains in the system but the user cannot log in.

## Glossary of Terms

**Administrator**: User with high-level access to manage the system

**Application**: Request submitted by a candidate for a job or scholarship

**Bundle**: Package of multiple course levels offered at a discounted price

**Dashboard**: Main screen showing system overview and statistics

**Eligibility**: Whether a student can attend classes based on payment status

**Enrollment**: Student registration for a language course

**Gateway**: Payment processor (Pesapal, M-Pesa, Flutterwave)

**Installment**: Partial payment (e.g., 50% now, 50% later)

**Module**: Different sections of the system (Jobs, Scholarships, Courses, etc.)

**Partner**: Organization that collaborates with AfriThrive Global

**Permission**: Authorization to perform specific actions in the system

**Reference Number**: Unique identifier for transactions or applications

**Role**: Set of permissions assigned to users

**Session**: Time period for a course timetable (e.g., Spring 2026)

**Status**: Current state of a record (Active, Pending, Completed, etc.)

**Track**: Schedule option for courses (Morning, Evening, Weekend)

**Timetable**: Schedule of when courses are offered
